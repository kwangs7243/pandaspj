# 개인 소비 데이터 분석 시스템  
## 기능 요구서 v2 (재사용 중심 설계 문서)

---

## 1. 문서 목적

본 문서는 개인 소비 데이터 분석 시스템의 개발 방향을 정의하는 상위 요구사항 문서이다.

이 시스템은 단순 기능 나열이 아닌,
재사용 가능한 분석 구조를 기반으로 다양한 분석 결과를 만들어낼 수 있는 모델을 목표로 한다.

---

## 2. 핵심 개발 방향

- 기능 나열이 아닌 조합 가능한 구조
- 조회 결과를 다른 분석의 입력으로 재사용 가능
- 모든 결과는 DataFrame / Series 형태 유지
- 전처리와 분석 로직 분리

---

## 3. 시스템 구성

### DataAnalyzer
- CSV 로드
- 데이터 전처리
- 문제 데이터 분리
- 분석용 데이터 반환

### ExpenseAnalyzer
- 조건 조회
- 요약 / 집계
- 순위 계산
- 비교 분석
- 통계 계산
- 시각화용 데이터 생성

---

## 4. 전처리 요구사항

### 날짜 처리
- 다양한 형식 지원
- datetime 변환
- year, month 컬럼 생성

### 타입 처리
- income/refund → 수입
- expense → 지출

### 카테고리 처리
- 공백 제거
- 표준화

### 금액 처리
- 쉼표 제거
- 숫자 변환

### 문제 데이터 분리
- 날짜 오류
- 타입 오류
- 금액 오류

---

## 5. 분석 모델 요구사항

### 핵심 개념
분석은 다음 흐름으로 조합 가능해야 한다:

filter → summary → compare → stat

---

## 6. 조회 기능

- 연/월 조회
- 기간 조회
- 타입 조회
- 카테고리 조회
- 금액 기준 조회
- 키워드 조회

※ 조회 결과는 반드시 다른 기능에 재사용 가능해야 한다

---

## 7. 요약 기능

- 전체 요약 (수입 / 지출 / 순이익)
- 월별 요약
- 카테고리별 요약
- 타입별 요약
- 카테고리별 건수

---

## 8. 순위 기능

- 타입별 상위 N
- 카테고리별 상위 N
- 전체 상위 N

---

## 9. 비교 기능

- 월 간 비교
- 카테고리 기준 비교
- 증감 / 증감률 계산

---

## 10. 통계 기능

- 카테고리별 평균
- 월별 평균
- 카테고리 비중

---

## 11. 확장 목표

- 시각화 연결
- API 구조 확장
- UI 연동 가능 구조 유지

---

## 12. 핵심 설계 철학

이 시스템은 기능 모음이 아니라 분석 엔진이다.

각 기능은 독립적으로 존재하지만,
반드시 다른 기능과 조합 가능해야 한다.
