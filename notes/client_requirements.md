# 개인 소비 데이터 분석 시스템  
## 기능 요구서 (클라이언트 제공 문서)

---

## 1. 문서 목적

본 문서는 개발자에게 전달하는 **기능 요구사항 문서**이다.  
개발자는 제공된 원본 CSV 파일을 기반으로 데이터 전처리, 조회, 집계, 비교, 통계, 시각화 기능을 구현해야 한다.

이 문서는 실행 파일이 아니며, 개발 진행 중 항상 참고해야 하는 기준 문서이다.

---

## 2. 프로젝트 개요

당사는 개인 소비 데이터를 정리하고 분석할 수 있는 간단한 분석 도구를 필요로 한다.  
다만 실제 입력 데이터는 정제되지 않은 상태로 들어오므로, 단순 조회 기능만으로는 충분하지 않다.

개발 결과물은 다음 흐름을 만족해야 한다.

1. CSV 파일 로드
2. 데이터 전처리
3. 분석 가능한 데이터셋 생성
4. 조회 / 요약 / 비교 / 통계 기능 제공
5. 필요 시 시각화 기능 제공

---

## 3. 제공 파일

### 3-1. 원본 데이터 파일
- 파일명: `messy_expense_client.csv`

### 3-2. 데이터 설명
원본 데이터는 가계부 성격의 소비 내역이며, 다음과 같은 문제가 포함될 수 있다.

- 날짜 형식이 통일되어 있지 않음
- 금액에 쉼표가 포함되어 있음
- 수입/지출 값이 일관되지 않음
- 카테고리에 공백, 대소문자 혼합이 존재함
- 일부 행은 잘못된 값일 수 있음

즉, **분석 기능 이전에 전처리 기능이 반드시 선행되어야 한다.**

# 6. 상세 기능 요구사항

---

## 6-1. 파일 로드 기능

### 기능 설명
CSV 파일을 불러와 DataFrame 형태로 저장한다.

### 요구사항
- UTF-8-SIG 인코딩 대응
- 결측치 표현(`not_available` 등)을 처리할 수 있어야 함
- 파일 로드 실패 시 적절히 예외를 확인할 수 있어야 함

### 기대 결과
- 원본 데이터가 DataFrame으로 적재됨

---

## 6-2. 데이터 전처리 기능

### 기능 설명
원본 데이터를 분석 가능한 형태로 정리한다.

### 세부 요구사항

#### 1) 컬럼명 정리
- 컬럼명을 일관된 이름으로 관리할 수 있어야 함
- 필요 시 원본 컬럼과 전처리 컬럼을 구분해도 무방함

#### 2) 날짜 처리
- 다양한 날짜 구분자(`-`, `/`, `.` 등)를 처리할 수 있어야 함
- datetime 타입으로 변환해야 함
- 변환 실패 데이터는 `NaT` 처리
- `year`, `month` 컬럼 생성 필요

#### 3) type 처리
다음 규칙을 적용한다.

- `income` → `수입`
- `refund` → `수입`
- `expense` → `지출`
- 그 외 값은 유효하지 않은 값으로 처리

#### 4) category 처리
- 앞뒤 공백 제거
- 대소문자 정리
- 필요 시 표준 카테고리명으로 매핑

예:
- `FOOD` → `식비`
- ` cafe ` → `카페`

#### 5) amount 처리
- 쉼표 제거
- 숫자형 변환
- 변환 실패 시 `NaN` 처리

#### 6) 유효하지 않은 행 분리
다음 조건 중 하나라도 해당하면 문제 행으로 분리 가능해야 함

- 날짜 변환 실패
- type 정리 실패
- amount 숫자 변환 실패

### 기대 결과
- 분석 가능한 정제 데이터 확보
- 문제 데이터만 따로 확인 가능

---

## 6-3. 조회 기능

### 기능 설명
사용자가 조건에 따라 데이터를 조회할 수 있어야 한다.

### 필수 기능 목록

#### 1) 연/월 조회
- 특정 연도와 월에 해당하는 데이터 조회

예시 메서드:
- `filter_by_year_month(year, month)`

#### 2) 기간 조회
- 시작일 ~ 종료일 사이 데이터 조회

예시 메서드:
- `filter_by_date_range(start_date, end_date)`

#### 3) 타입 조회
- 수입만 보기
- 지출만 보기

예시 메서드:
- `filter_by_type(type_name)`

#### 4) 카테고리 조회
- 특정 카테고리만 조회

예시 메서드:
- `filter_by_category(category_name)`

#### 5) 최소 금액 기준 조회
- 특정 금액 이상 데이터만 조회

예시 메서드:
- `filter_by_min_amount(min_amount)`

#### 6) 키워드 조회
- content 컬럼 기준 문자열 검색

예시 메서드:
- `filter_by_keyword(keyword)`

---

## 6-4. 요약 기능

### 기능 설명
데이터를 집계하여 요약 정보를 제공해야 한다.

### 필수 기능 목록

#### 1) 전체 요약
다음을 반환해야 한다.

- 총 수입
- 총 지출
- 순이익

예시 메서드:
- `summary_total()`

#### 2) 특정 월 요약
특정 연월의 수입/지출/순이익 요약

예시 메서드:
- `summary_by_year_month(year, month)`

#### 3) 월별 전체 요약표
월별 수입/지출/순이익을 한 번에 보여주는 표

예시 메서드:
- `summary_by_month()`

#### 4) 타입별 요약
수입 합계 / 지출 합계

예시 메서드:
- `summary_by_type()`

#### 5) 카테고리별 요약
카테고리별 총합 계산  
옵션에 따라 수입만 / 지출만도 가능해야 함

예시 메서드:
- `summary_by_category(type_name=None)`

#### 6) 카테고리별 건수 요약
각 카테고리의 데이터 개수 반환

예시 메서드:
- `summary_count_by_category()`

## 6-5. 순위 기능

### 기능 설명
금액 기준 상위 데이터를 확인할 수 있어야 한다.

### 필수 기능 목록

#### 1) 타입별 상위 N개
수입 또는 지출에서 상위 N개 조회

예시 메서드:
- `get_top_n_by_type(type_name, n)`

#### 2) 카테고리별 상위 N개
특정 카테고리 안에서 상위 N개 조회

예시 메서드:
- `get_top_n_by_category(category_name, n)`

#### 3) 전체 상위 N개
전체 데이터에서 금액 기준 상위 N개 조회

예시 메서드:
- `get_top_n_overall(n)`

## 6-6. 비교 기능

### 기능 설명
기간 또는 월 단위 비교 결과를 제공해야 한다.

### 필수 기능 목록

#### 1) 두 달 비교
두 달의 수입/지출/순이익 비교

예시 메서드:
- `compare_months(year1, month1, year2, month2)`

#### 2) 카테고리 월 비교
특정 카테고리에 대해 두 달 또는 여러 달 비교

예시 메서드:
- `compare_category_between_months(category, year1, month1, year2, month2)`
